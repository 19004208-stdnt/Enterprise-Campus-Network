Project Overview

This project demonstrates the design and implementation of a scalable enterprise campus network using Cisco Packet Tracer. The network follows Cisco's three-tier hierarchical architecture consisting of the Core, Distribution, and Access layers.


The simulated campus environment includes an ISP cloud, Cisco 2911 edge router, Cisco ISA-3000 firewall, Cisco 3650 core switch, Cisco 3560 distribution switches, Cisco 2960 access switches, department PCs, printers, and a planned data centre services network.


The project is designed to demonstrate practical skills required for roles such as IT Support Technician, NOC Technician, Junior Network Engineer, IT Infrastructure Technician, and Server Support Engineer.

Network Architecture

The network uses a structured enterprise design where each layer has a clear role: the Core layer provides high-speed backbone connectivity, the Distribution layer aggregates departmental access switches, and the Access layer connects end-user devices.

ISP Cloud → Edge Router → ISA Firewall → Core Switch → Distribution Switches → Access Switches → End Devices

WAN / Internet Edge

The ISP Cloud and Cisco 2911 router simulate internet-facing connectivity and routing toward the firewall.

Security Boundary

The ISA-3000 firewall acts as the security layer between the external network and internal campus infrastructure.

Core Layer

The Cisco 3650 core switch provides VLAN gateways and supports inter-VLAN routing.

Access Layer

Cisco 2960 switches connect users, printers, and department devices to the correct VLANs.

Implementation Phases

Phase 1: Physical Topology

Designed the campus topology using ISP, router, firewall, core switch, distribution switches, access switches, PCs, and printers.

Phase 2: VLAN Design

Created VLANs for Administration, ICT, Engineering, Science, Library, Residence, and Data Centre.

Phase 3: Trunk Configuration

Configured 802.1Q trunk links between core, distribution, and access layer switches.

Phase 4: Inter-VLAN Routing

Configured VLAN gateway interfaces on the core switch to route between departments.

Phase 5: WAN and Firewall

Integrated edge routing and firewall interfaces to simulate an enterprise internet edge.

Phase 6: Verification

Validated the network using Cisco IOS show commands, MAC tables, CDP neighbors, and connectivity testing.

Challenges and Solutions

Challenge 1: Interface Naming Differences

Some Cisco switches used different interface naming formats. For example, the core switch used GigabitEthernet1/0/x, while the 3560 and 2960 switches used FastEthernet0/x and GigabitEthernet0/x.

Solution

I used verification commands such as show ip interface brief and show cdp neighbors to identify the correct physical ports before applying trunk or access-port configurations.

Challenge 2: Trunk Links Not Appearing

Some trunk links did not immediately appear under show interfaces trunk because only one side of the connection was configured.

Solution

I configured both ends of the switch links as trunk ports and allowed the required VLANs. This enabled VLAN traffic to pass between the core, distribution, and access layers.

Challenge 3: Firewall Interface Configuration

The ISA-3000 firewall used different interface names from common ASA examples, so commands like interface gigabitEthernet0/0 were not valid.

Solution

I used show interface ip brief to identify the correct firewall interfaces such as GigabitEthernet1/1 and GigabitEthernet1/2, then configured inside and outside connections.

Challenge 4: GitHub Pages Image Paths

After deployment, screenshots were not displaying because the project screenshots were stored in a nested folder while the index.html file was stored in the repository root.

Solution

I updated the image paths to point to the correct GitHub Pages location: /Enterprise-Campus-Network/Enterprise-Campus-Network/screenshots/filename.png.
